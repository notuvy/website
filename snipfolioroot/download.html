Android App

And now the Android App for SnipFolio is available for download. Because it is an alpha version, it is a direct download rather than from the Android marketplace.

You can click on this download link to get it. After the download is complete (notification in the status bar), you can click on the download to install it.

NOTE: You will need to enable your device to install non-Market applications. This is done at:
    Menu > Settings > Applications > Unknown sources

Desktop Application

A preliminary version of the SnipFolio desktop application is now available for download. Get it from the link below.

But first, a few notes:

  1. SnipFolio is available right now by invitation only, so you will be asked for a username/password when you click the "Download" link.
  2. SnipFolio is a Java application, so it should work on all operating systems because most have Java installed by default. (OS-specific install instructions are below.)
  3. This version is an experimental, pre-release version. It does contain some known defects. Newer versions will have them fixed.
  4. This demo version does not encrypt centrally stored data. Do NOT store sensitive data to centrally stored lists. (It is safe to store them in local lists, however.)
  5. Because of possible instability with the early version, no guarantee can be made of the continuous availability of the centrally stored lists (but every effort is made to keep data safe).

With all that out of the way, you can now Download SnipFolio!

And once you have installed it (minimal instructions below), here is some basic documentation on getting started.

Once you have downloaded the "jar" file, you need to know what to do with it. Here are the instructions.

Windows and Mac
Save the file to your Desktop. Double-click on the icon to launch the application.
You can move it to a different location (double-click will still launch it).
Linux
Save the file anywhere in your file system.
Lauch it with the command: java -jar snipfolio-desktop-1.0.0-full.jar
(Of course, use the correct version number in the command.)
