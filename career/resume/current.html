EMPLOYMENT HISTORY

Founder and Chief Architect SnipFolio LLC.,
Sunnyvale, CA		 October 2011 - present

SnipFolio moves copy-and-paste to the cloud. SnipFolio is a product that I took from an idea to general release, with both a desktop application and an Android app. The application is publicly available, highly rated, and has a growing install base.

SnipFolio started as one of several personal projects years earlier, but it was not being actively developed. After leaving Local Corporation, I decided to try to commercialize one of my projects. I decided to pursue SnipFolio because I did not see any other solutions for a cloud-based clipboard manager. I formed the company and filed the patent covering the unique core functionality.

The design work involved several aspects: data structures, client/server communication, UI design, scalability, and security. I am responsible for all aspects of engineering, business development, and marketing.

Java was chosen for implementation of the desktop client so that it would be available on all operating systems. (If more resources were available, native applications would have been explored. Java provides access to the system clipboard sufficient for a general client, but native APIs might provide more robust access.) The choice of Java also meant that a large portion of the codebase could be reused in the implementation of an Android mobile app.

The REST webservice paradigm was chosen to handle data exchange due to its simplicity and flexibility. The specific framework choice was Restlet, for its simplicity.

All server communication was designed to be HTTP web services because this is the most general security solution (it satisfies requirements for operation behind a secure firewall). Because this necessitates periodic polling, clever algorithms needed to be defined for efficient and effective data synchronization.

Although UI design is not my primary strength, a good level of competence was required to create a public application. The main challenge was to include powerful features but still keep the appearance simple. Another major challenge was to design both the desktop and mobile applications with an (almost) identical look-and-feel, a requirement for a smooth user experience.

A cache for server data was implemented to optimize performance. A custom design was done to conform to expected and observed usage.

Several aspects of security were designed and implemented. HTTPS is used for all server communication to ensure all network data exchange is protected (with no data exposed in the GET request). Sensitive data are stored with encryption (PBEWithMD5AndDES) so that information is protected even if the system is compromised. User authentication is handled with OAuth2 to leverage existing authentication providers both as a user convenience and as a security feature.

The IT infrastructure choice was to use Amazon EC2 for its configuration flexibility and convenience. The whole system currently operates in a single micro instance, but can be easily expanded to a larger system due to its modularity. The web server front end is handled by apache, and the web application is deployed to a tomcat instance (mod_proxy connected). This configuration enables expansion when needed.
