SnipFolio Summary: Murali Krishnan

Founder and Chief Architect, October 2011 - present

Overview

After leaving Local Corporation, I decided to work on my personal projects. I spent some time on my open source projects, but also on some unreleased private projects.

I had been working on a clipboard manager for a few years. I investigated that available clipboard managers on the market and found that none provided data sharing across computers, so I determined this to be a market opportunity.

Framework Design and Architecture

Design Decisions

Java was chosen to be the language for client development. This decision was made because Java supports access to clipboard data, and it is a cross-platform language, so that a single client can be developed for multiple operating systems.

Data Representation

It was decided to use XML for all data exchange.

REST Server

Having a centralized server for centralized data storage would be necessary.
